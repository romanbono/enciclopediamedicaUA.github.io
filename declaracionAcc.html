Declaración de Accesibilidad

Nos comprometemos a garantizar que el sitio web de la Enciclopedia Médica Universidad de Alicante sea accesible para el mayor número posible de usuarios, independientemente de sus capacidades. A continuación, describimos las características implementadas para asegurar la accesibilidad del sitio:

1. Identificación del idioma principal de la página

El idioma principal de cada página está claramente identificado utilizando el atributo lang="es" en la etiqueta <html>. Esto asegura que los lectores de pantalla puedan interpretar correctamente el contenido y su pronunciación adecuada en español.

2. Etiquetado semántico

El sitio web utiliza correctamente el etiquetado semántico para garantizar una mejor comprensión del contenido. Las etiquetas <header>, <nav>, <main>, <footer> se emplean adecuadamente para describir las diferentes secciones de cada página. Esto facilita la navegación.

3. Texto alternativo para las imágenes

Todas las imágenes incluyen texto alternativo con el atributo altque describe su contenido. Además con el atributo title en las etiquetas <iframe>, donde se muestran los vídeos, también se incluye texto alternativo descriptivo para estos.

4. Mecanismos de navegación

Se han implementado mecanismos claros de navegación para facilitar el acceso a todas las secciones del sitio. Esto incluye:

  • El sitio incluye un mapa del sitio web, que proporciona una vista completa de todas las secciones y páginas disponibles.
  • Se ha implementado una ruta de migas de pan en cada página que muestra la jerarquía del sitio y permite volver fácilmente a secciones anteriores.
  • Enlaces adicionales en cada página para facilitar la vuelta al índice o a la página principal.
  • Un navegador de artículos alfabético con enlaces a los diferentes índices de letra
  • Enlaces adicionales en cada página para facilitar la vuelta al índice o a la página principal.
  • Un menú de navegación para temas específicos dentro de los artículos

5. Contraste de color.

Todos los colores empleados para dar estilo a la página cumplen rigurosamente con las directrices del estándar WCAG sobre el contraste de color.

Nivel de contraste empleado:

  • Para texto normal (por debajo de 18 puntos), el contraste mínimo es de 4.5:1.
  • Para texto grande (18 puntos o más, o 14 puntos en negrita), el contraste mínimo es de 3:1.

6. Resaltado del foco en los elementos de interacción:

  • El resaltado del foco en elementos interactivos (como botones, enlaces y campos de formulario) es esencial para mejorar la accesibilidad y usabilidad.
  • Todos los elementos interactivos muestran un indicador visible cuando reciben el foco del teclado, ayudando a los usuarios que navegan sin ratón.
  • El foco generalmente se representa mediante un borde, sombra o cambio de color claro y visible.
  • Se ha evitado la eliminación del outline predeterminado de los navegadores a menos que se proporcione una alternativa visible y accesible.
  • Un contraste adecuado del resaltado del foco frente al fondo es crucial para que sea fácilmente perceptible.

7. USo de diferentes estilos de página que mejoran las accesibilidad:

Se han empleado distintos estilos de página pensando en ofrecer la mejor experiencia posible a todos los usuarios con capacidades. Para ello se han diseñado los siguientes estilos de página, que se pueden elegir desde el selector de estilos del navegador.

  1. Modo estándar: Permite al usuario disfrutar de la versión general para todo el mundo, pensada para personas que no tienen discapacidad de ningún tipo.
  2. Modo Noche: Similar al modo anterior, pero pensado para utilizarse en habitáculos con luz escasa, ayudar a descansar la vista y mejorar el ahorro de batería.
  3. Modo Impresión: Permite la impresión estilo periódico de la página web, aportando legibilidad y claridad al documento impreso.
  4. Modo Alto Contraste: Este modo está pensado para personas con baja visión que necesitan un contraste mayor de color para percibir correctamente los elementos de la página.
  5. Modo Letra Aumentada: Este modo está pensado para personas con baja visión que necesitan un mayor tamaño de letra para percibir correctamente los elementos de la página.
  6. Modo Accesible: Este modo combina los dos anteriores para brindar al usuario con problemas de visión una experiencia completa y adaptada a sus capacidades visuales.

8. Otras características de accesibilidad

-Tablas accesibles:
Las tablas están correctamente etiquetadas con encabezados que permiten a los lectores de pantalla identificar las columnas de manera eficaz.
-Enlaces descriptivos:
Los enlaces del sitio son descriptivos y proporcionan información clara sobre su destino.
-Descripción de siglas y abrebiaturas:
Las siglas en nuesta web estan descritas con el atributo title dentro de la etiqueta <abbr> para aclrar su significado y evitar confusiones.
-Correcto uso de encabezados:
Haciendo un correcto uso de los encabezados (<h1><h2><h3>...), definimos una estructura organizada y facilmente navegable.

Si tiene algún problema de accesibilidad o desea sugerir mejoras, no dude en contactarnos a través del formulario de contacto.
